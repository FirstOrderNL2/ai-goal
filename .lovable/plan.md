

# AI Reasoning System Analysis & Improvement Plan

## Current State Assessment

### Database Reality (1,483 predictions)
- **Only 26 (1.8%) have ai_reasoning** — the vast majority were generated by an older system
- **Only 26 have predicted scores** — 98% lack scoreline predictions
- **1,411 (95%) have xG = 0.0** — broken/missing expected goals
- **1,467 (99%) have BTTS = "no"** — default value, never computed
- **1,443 (97%) have over_under = "under"** — default value, never computed
- **match_context: 0 injuries, 0 lineups, 0 weather** — only h2h_summary is populated (29 rows), structured fields are never written to
- Post-match reviews: 102 exist, avg accuracy self-score 78/100

---

## ✅ What is Working

1. **The new prediction engine (generate-ai-prediction)** uses Gemini 2.5 Pro with high-effort reasoning, tool calling for structured output, and demands fact-based justifications — this is architecturally sound
2. **Data gathering is comprehensive**: form (overall + home/away splits), H2H, goal stats (avg scored/conceded, BTTS rate, clean sheets), odds, and live web context are all fetched in parallel
3. **Probability normalization** ensures home_win + draw + away_win = 1.0
4. **Learning feedback loop**: past reviews with accuracy scores are fed back into the prompt
5. **Caching layer** (match_context, 6-hour TTL) prevents redundant scraping

---

## ❌ What is Not Working

1. **95% of predictions are empty shells** — old predictions have no xG, no scores, no BTTS, no reasoning. The backfill_xg function exists but only estimates xG from probabilities (circular logic — asking AI to guess xG from win% instead of from actual match data)

2. **match_context structured fields are NEVER populated** — `cacheContext()` only writes to `h2h_summary` as a raw text blob. The `injuries_home`, `injuries_away`, `lineup_home`, `lineup_away`, `suspensions`, `weather`, `news_items` columns are always empty arrays/null. The MatchContextCard UI checks these fields but they're always empty, so it never renders

3. **Batch function uses a weaker prompt** — no `reasoning: { effort: "high" }`, no learning context from past reviews, shorter system prompt, single `reasoning` field instead of the 4 separate reasoning fields (winner/btts/over_under/key_factors)

4. **model_confidence = highest probability** — many old predictions set confidence equal to the win probability (e.g., Brazil 92.3% → confidence 92.3%). This conflates "how dominant is the favorite" with "how reliable is this prediction"

5. **Over/Under and BTTS are defaults** — the database defaults are `under` and `no`, and old predictions never overwrote them

---

## ⚠️ Weak Reasoning Areas

1. **No mathematical model** — there is zero statistical computation of probabilities. The system sends raw form strings to the AI and asks it to output probabilities. The AI is doing all the math intuitively, which means:
   - No Poisson distribution for goal probabilities
   - No Elo rating system
   - No weighted form (recent matches should matter more)
   - Probabilities are AI vibes, not computed

2. **Odds are underused** — odds are shown to the AI as a single line (`Odds: Home 1.8, Draw 3.5, Away 4.2`) but there's no:
   - Conversion of odds to implied probabilities
   - Comparison of model vs market
   - Value detection (model says 60% but odds imply 45%)

3. **xG data is rarely available** — the system fetches `xg_home`/`xg_away` from match records but most matches don't have xG data. The field is shown in the prompt when available but there's no fallback estimation

4. **Form recency is equal** — all 5 form matches are treated equally. A match from 2 weeks ago has the same weight as one from 3 months ago

5. **No league context** — the AI doesn't know if a team is 1st vs 20th in the table. League position/points are never fetched or provided

6. **Scraped data quality is poor** — Firecrawl results are raw markdown dumps (up to 4000 chars each). The AI synthesis step uses a flash model and produces a text blob that's cached as `h2h_summary` rather than parsed into structured fields

7. **Self-scoring bias** — the AI gives itself an average accuracy of 78/100, which seems high. The AI is rating its own predictions, which creates a feedback loop of overconfidence

## 🔧 Improvements Plan

### Phase 1: Fix Data Pipeline (Critical)

**1a. Parse match_context into structured fields**
- In `fetch-match-context/index.ts`, after the AI synthesis step, use tool calling to extract structured data (injuries array, lineup array, suspensions array, weather string, news items) and write to the proper columns instead of dumping everything into `h2h_summary`

**1b. Add pre-computed statistics to the prediction prompt**
- Calculate Poisson-based goal probabilities from avg goals scored/conceded before sending to AI
- Convert odds to implied probabilities and include the comparison
- Add league position data (requires new table or API call)

**1c. Backfill existing predictions properly**
- Create a proper backfill function that re-runs the new prediction engine on old matches instead of the weak xG-from-probability approach

### Phase 2: Strengthen Reasoning Engine

**2a. Add a statistical pre-computation layer in generate-ai-prediction**
```
Before calling the AI, compute:
- Poisson expected goals: home_attack × away_defense_weakness
- Implied probabilities from odds (1/odds normalized)
- Model vs market delta
- Form-weighted score (recent = higher weight)
- Over 2.5 probability from Poisson
- BTTS probability from scoring/conceding rates
```
These computed values should be included in the prompt so the AI has mathematical anchors rather than guessing.

**2b. Align batch-generate-predictions with generate-ai-prediction**
- Add `reasoning: { effort: "high" }` to batch function
- Use the same 4-field reasoning schema (winner, btts, over_under, key_factors)
- Include the learning context from past reviews
- Include pre-computed stats

**2c. Add confidence calibration**
- Confidence should reflect data quality (how much form data exists, whether odds are available, whether injuries are known) — not prediction strength
- Define a formula: `confidence = 0.2*(form_matches/5) + 0.2*(has_odds) + 0.2*(has_h2h) + 0.2*(has_injuries) + 0.2*(has_league_position)`

### Phase 3: Improve Explainability & UI

**3a. Show computed stats alongside AI reasoning**
- Display the mathematical inputs (Poisson xG, implied odds, form-weighted stats) on the match detail page so users can see what data the AI used
- Add a "Data Quality" indicator showing what data was available vs missing

**3b. Add market comparison section**
- Show "AI says 55% home win, market says 48% → potential value on home win"

**3c. Fix MatchContextCard to actually display data**
- Currently it checks empty arrays and renders nothing — fix the data pipeline first (Phase 1a)

### Phase 4: Consistency & Validation

**4a. Add prediction validation**
- Ensure predicted_score matches over_under verdict (score 2-1 = 3 goals → should be "over")
- Ensure BTTS matches predicted score (1-0 → BTTS should be "no")
- Flag and reject inconsistent predictions

**4b. Independent accuracy scoring**
- Post-match review should compare actual outcomes to predicted outcomes mathematically (Brier score, exact score hit rate) rather than letting the AI self-grade
